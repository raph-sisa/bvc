### GitHub Workflow for Participants

This workshop uses a **two-repository approach** for optimal learning:

**1. Workshop Repository (This Repo - Forked by Participants)**
- **Purpose:** Access materials and contribute resources to community
- **Participant Action:** Fork this repo
- **Usage:** Submit Pull Requests to share resources in `community-resources/`
- **Visibility:** Public (required for collaboration)

**2. Assignment Repository (Based on participant-template/)**
- **Purpose:** Personal workspace for assignments and capstone
- **Participant Action:** Create from template (or fork `participant-template/`)
- **Usage:** Complete weekly assignments, build portfolio
- **Visibility:** Participant's choice (private or public)

See `CONTRIBUTING.md` for detailed PR guidelines and `week-1/Week1-Assignment-Instructions.md` for setup instructions.
